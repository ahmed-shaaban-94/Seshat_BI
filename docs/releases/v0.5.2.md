# Seshat BI -- v0.5.2 patch release candidate

This patch candidate records the changes merged to `main` after the frozen
v0.5.1 release note. It does not rewrite the historical claims captured in
`docs/releases/v0.5.md` or `docs/releases/v0.5.1.md`.

## What this patch adds

- **`seshat dashboard`** (issue #358, closing the deferred dashboard scope #359
  #360 #361): writes a self-contained, static HTML readiness view of the
  workspace and can auto-open it. The page has a Home section with portfolio-level
  KPIs and one card per table showing that table's position on the seven-stage
  readiness track, rendered from the recomputed `readiness-status.yaml`. It ships
  as a single HTML file -- no server, no external assets, theme CSS inlined -- and
  every interpolated value is HTML-escaped. The renderer fails safe on a `None`
  stage or a missing source path rather than raising. This is a read-only view of
  already-computed readiness state; it computes no new confidence and grants no
  approval.

## What this patch fixes

- **Three credential-leak paths in Dagster-adapter and portfolio redaction**
  (issues #362, PR #364; building on #363): a *reformatted*, schemeless driver
  error that named only the host/user components of a `DATABASE_URL`-shaped secret
  -- with neither the verbatim DSN nor a `scheme://` present -- could survive the
  whole-value replace and reach an evidence surface. The redactors now decompose
  the URI and scrub each component (host/user/password/dbname) on its own.
- **Over-redaction of fixed-vocabulary config words** (issue #357, PR #363): the
  Dagster adapter now derives its secret environment values from an explicit
  POSITIVE key set (the `ANALYTICS_DB_*` credential subset plus `DATABASE_URL`)
  rather than a prefix scan, so a non-secret config knob such as
  `ENGINE=postgres` is no longer shredded into a partial redaction, while genuine
  credentials stay scrubbed at any length.
- **`seshat-init` workspace writers and `init-project` layout** (issues #349
  #350 #351 #352, PR #356): the workspace scaffold writers are hardened and the
  generated project layout is aligned.
- **A time-of-check-to-time-of-use race in `scaffold-source`** (issue #345, PR
  #355): the per-file `_write_if_absent` write path is closed. This is the
  follow-up that v0.5.1's note recorded as a deferred, tracked item.
- **Workspace `.env` in the Dagster command family** (issue #348, PR #354): the
  Dagster commands now read the workspace `.env` for engine selection and
  connection resolution, matching how `validate` and `value-check` already do,
  so credentials placed in the gitignored `.env` are honored.

## What this patch changes internally

- **One shared URI-redaction core** (issue #365, PR #366): the URI-component
  decomposition and the fragment-replace helper -- previously copied into three
  modules by the #362 fix -- are extracted into a single stdlib-only module,
  `seshat/redaction_core.py`, imported by the dbt, Dagster-adapter, and portfolio
  redactors. Each caller keeps its own replacement token. This is a pure refactor
  with no behavior change; the full redaction test suite stays green with tokens
  unchanged.

## Honest boundaries

- The dashboard is a read-only rendering of the already-recomputed
  `readiness-status.yaml`; it fabricates no evidence, approvals, or readiness
  confidence, and it is not a governance gate.
- The redaction fixes narrow real leak paths but do not claim to redact every
  conceivable driver-error reformatting; the scrubbers fail closed (drop to a
  generic boundary error) when the redactor itself cannot guarantee a clean
  string.
- The `#345` TOCTOU fix closes the deterministic path-safety cases in the
  scaffolder's per-file write; a hostile concurrent process inside the workspace
  remains an operating-environment concern, not a kit guarantee.
- This candidate does not itself create a tag, publish to PyPI, create a GitHub
  Release, or submit to public Claude, Codex, or OpenAI catalogs. External
  marketplace publication remains a separate owner action.
