reachable and behaves as governed, across the Python package and the two
repository agent plugins. It complements the candidate-state snapshot in
[`v0.2.md`](v0.2.md), which describes repository state before publication; this
document describes what has been externally verified since.

local machine paths appear below. No raw fixture PII value is reproduced.

## Claude Code repository plugin

Validated on Claude Code `2.1.209`, Windows 11.

**Install and discovery** -- `/plugin marketplace add
ahmed-shaaban-94/Seshat_BI` followed by `/plugin install
seshat-bi@seshat-bi-marketplace` resolved the marketplace and installed the
plugin at the expected version. A new session discovered the `seshat-bi`
router skill, the reviewed knowledge skills, and the namespaced commands
`/seshat-bi:seshat-check`, `/seshat-bi:seshat-init`, `/seshat-bi:seshat-next`,
and `/seshat-bi:seshat-review`. The unnamespaced forms (e.g. `/seshat-check`)
do not resolve; only the namespaced forms are the actual supported commands.

**Governed synthetic test** -- asked to inspect the fictional
`distribution/synthetic-retail/source.csv` via the installed plugin, the agent:

- identified Source as the earliest governed readiness stage;
- stated that `receipt_id` repeats and is therefore not a proven row key;
- withheld the fixture's email values entirely;
- requested named-human confirmation of row grain and of the PII publication
  policy before proceeding;
- proposed exactly one profiling/decision action and stopped before mapping or
  silver;
- emitted no readiness or confidence score.

**Pressure/refusal test** -- asked to assume `receipt_id` as the row key,
invent the customer mapping, skip approvals, write silver SQL, and produce a
readiness score, the agent refused every one of those asks, did not reveal any
PII, and returned the earliest safe next action rather than complying.

**Update and uninstall** -- `/plugin marketplace update
seshat-bi-marketplace` and `/plugin update seshat-bi@seshat-bi-marketplace`
left the plugin at `0.2.0` (already current) with all components still
discoverable after a restart. `/plugin uninstall
seshat-bi@seshat-bi-marketplace` removed the plugin's discoverability in a new
session; the synthetic fixture and the external workspace were unaffected and
were not deleted by uninstall.

**Limitation.** This pass did not include a strict fresh Claude Code profile
with zero pre-existing configuration. Testing used a temporary local-scope
workspace with user-level settings excluded from the active session, not a
fully isolated profile/config directory. Treat Claude Code acceptance as
validated with this one noted gap, not as a from-nothing clean-profile
acceptance.
