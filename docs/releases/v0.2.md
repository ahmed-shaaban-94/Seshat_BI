# Seshat BI -- v0.2 Public Beta release candidate

This release candidate records the repository state prepared from
`48e71f73d5f6e69a214f8e05c3b34517e8acaa68` on 2026-07-13. It is a repository
candidate only: it does not prove PyPI, GitHub Release, Claude catalog, Codex
catalog, or OpenAI public-plugin availability. Each of those actions requires its
own named-owner approval and external evidence.

## Product identity

Seshat BI remains an **agent-first BI readiness system**. It requires evidence and
named-human approvals at readiness gates; it does not turn an agent recommendation,
a version number, or a release-candidate audit into an approval.

The `seshat` command is now the primary brand alias, while `retail` remains a
compatibility alias; both dispatch to the same CLI entry point. Evidence:
`ca0d76c` (roadmap M1).

## What this candidate adds since v0.1.0

- A public-beta distribution foundation: deterministic Claude and Codex knowledge
  bundles, version-projection auditing, candidate auditing, artifact inspection,
  release-workflow safeguards, and credential-free acceptance fixtures. Evidence:
  `48e71f7` (PR #266).
- A first-run workspace initializer, `seshat init-project`, which creates an
  empty governed workspace without silently overwriting a non-empty target.
  Evidence: `0f5f0b3` (spec 107, PR #217).
- A read-only readiness-status projection via `seshat status`; it reports
  committed readiness state without creating a pass or numeric score. Evidence:
  `132d6a8` (spec 109, PR #223).
- The agent ecosystem additions: offline readiness proof, review/SARIF output,
  a read-only MCP surface, readiness passports, extension packs, contributor
  surfaces, an agent-safety benchmark, and an offline readiness explorer.
  Evidence: `c105db8` (spec 120, PR #254).
- A business-knowledge interview, decision-store, and knowledge-contract
  foundation that keeps critical business decisions and their approvals
  reviewable. Evidence: `0aca21d` (spec 121, PR #257).
- Contract-driven portfolio discovery, governed dashboard-intelligence work, and
  generic KPI contract authoring support. Evidence: `64af33e` (PR #260),
  `88daf50` (PR #261), and `785299e` (spec 124, PR #264).

## Documentation and release discipline

The candidate includes the user install path, source-onboarding, mapping-review,
evidence-pack, and BI-delivery guides, plus the semver policy, changelog process,
and install smoke-test foundation. Evidence: `6138540`, `c966527`, `da1218f`,
`2070986`, `c361f54`, and `2719fb6` (PRs #218-#221 and #224).

## What remains deliberately outside this candidate

- No tag, GitHub Release, package-index publication, catalog publication, or
  OpenAI public-plugin submission has been performed.
- Public installation evidence for Python, the Claude repository flow, and the
  Codex repository flow remains unverified until owner-authorized external
  acceptance is recorded.
- The Power BI execution adapter (F016) remains deliberately deferred and
  execution-only; it is not unlocked by this candidate.

## Release criteria

The owner must review the exact candidate revision and generated artifact digests,
then make separate action-scoped decisions for tag creation, package publication,
GitHub Release publication, and any catalog or public-plugin submission. See
[`docs/operations/release-acceptance-checklist.md`](../operations/release-acceptance-checklist.md)
and [`docs/operations/versioning-policy.md`](../operations/versioning-policy.md).
