# Seshat BI -- v0.6.1 patch release candidate

This patch candidate records the single change merged to `main` after the
frozen v0.6.0 release note (`docs/releases/v0.6.md`). It does not rewrite the
historical claims captured there or in any earlier release note.

## Why this patch exists

The v0.6.0 tag is frozen by the repository's `v*` tag-immutability ruleset and
points at a commit whose packaged `seshat/cli/__init__.py` trips the release
audit (`scripts/inspect_release_artifacts.py`). Because a protected tag cannot
be moved, v0.6.0 can never publish to PyPI. This patch carries the audit fix
forward on a fresh, publishable version.

## What this patch fixes

- **Release audit false positive blocking publication** (PR #426): the
  `_postgres_target_label` docstring in `seshat/cli/__init__.py` contained a
  fabricated spoof example (`abc://u:s3cret@x`) used to teach why content-based
  credential detection is unsafe. That example matched the release inspector's
  credential-bearing-URL pattern and blocked the v0.6.0 publish. The example is
  rephrased in prose so it no longer forms a literal `scheme://user:pass@`
  token, preserving the teaching intent. The scanner regex is deliberately left
  strict -- no credential guard was weakened. The change is docstring-only and
  behavior-neutral (the redaction / target-label unit tests are unchanged and
  still pass); the built wheel and sdist score zero hits across all of the
  inspector's content patterns.

## Honest boundaries

- This is a **packaging-only patch**: it carries no functional code, schema, or
  CLI-behavior change beyond the docstring rewording. Every capability claim in
  `docs/releases/v0.6.md` continues to apply unchanged.
- This candidate does not itself create a tag, publish to PyPI, create a
  GitHub Release, or submit to public Claude, Codex, or OpenAI catalogs.
  External marketplace publication remains a separate owner action.
