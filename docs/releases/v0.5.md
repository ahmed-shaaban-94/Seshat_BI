# Seshat BI -- v0.5 release candidate

This candidate records the changes merged after the frozen v0.4.1 release note.
It does not rewrite the historical claims captured in `docs/releases/v0.4.md`
or `docs/releases/v0.4.1.md`.

## What this release adds

- **Exact, table-agnostic dbt parity evidence** (issues #324/#331, PRs #330 and
  #332): parity validation is class-driven instead of hardcoded to the worked
  example, and the approved source map's `gold_star.fact` section now declares
  the grain/business key (composite grains as an ordered list) and the additive
  money measures (an explicit empty set for factless facts). The declarations
  are bound into the digest-accepted execution plan (plan schema v2) and the
  evidence layer verifies the expected fact-subject set exactly -- the wrong
  relation, the wrong key column, or an undeclared numeric column can no longer
  reconcile parity.
- **The mapping stage guarantees its decision ledger** (issue #326, PR #333):
  the new `seshat mapping-mirror` verb materializes
  `mappings/<table>/unresolved-questions.md` when absent. A CLEARED stub is
  derived only from committed readiness state carrying a named-human,
  C4-shaped `mapping_ready` approval with evidence and no blockers; anything
  less yields an OPEN stub that keeps every downstream gate blocked.
- **Self-sufficient workspaces** (issue #325, PR #335): `seshat dbt init` and
  `seshat dagster init` materialize the governed projects from bundled package
  data, so a bare `pip install seshat-bi[dbt]` workspace gains full dbt and
  Dagster capability without cloning the development repository. Bundled
  templates are strictly table-neutral.
- **Committed-state GO signal** (issue #334, PR #336): the Dagster gate reads
  `UNCOMMITTED` for any gate mirror that is untracked or dirty against HEAD, so
  the silver build can never be permitted by a clearance outside audit history.
- **`kpi-contract-builder`** (PR #321): the governed KPI contract drafting verb,
  registered in the kit source and capability inventory.
- **Windows resilience fixes** (PR #327): UTF-8 console output and multi-table
  `seshat dbt validate`.

## Honest boundaries

- New required `gold_star.fact` tags are a fail-closed change: an existing
  dbt-path mapping without `business_key` / `additive_money_measures` blocks at
  `seshat dbt validate` with `DBT_FACT_SEMANTICS_MISSING` until the owner
  declares the tags and re-approves the mapping. Accepted v1 execution plans are
  refused with a re-plan message.
- Live dbt compile, build, test, and parity evidence still require a named live
  profile and remain `[PENDING LIVE PROFILE]` without one.
- Dagster schedules and sensors remain STOPPED until a named human starts them.
- `seshat dagster init` materializes the orchestration project but its `.venv`
  is created by the operator; the doctor names the exact install command.
- This candidate does not itself create a tag, publish to PyPI, create a GitHub
  Release, or submit to public Claude, Codex, or OpenAI catalogs.
