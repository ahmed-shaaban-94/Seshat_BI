## REJECT

### F6. Anomaly-Baseline Contracts (statistical exception flags)

`HORIZON` · **V4 / F4** · consistency: minor-tension · **INELIGIBLE — violates a hard principle**

**Why this verdict:** Ineligible as framed against the ML/statistical out-of-scope principle. Introducing a definition.kind: anomaly_baseline carrying std-dev / trailing-mean / window semantics into the metric-contract template puts statistical inference INTO the contract layer — that is the forecasting/ML creep the repo explicitly defers (out-of-scope: ML/forecasting; numeric confidence/scoring deferred until rules are owner-set). Mark ineligible; if ever revisited it needs an explicit ADR ruling that owner-parameterized statistics are in-scope.

**First step:** None - reject as scoped; revisit only behind an explicit ADR deciding whether owner-parameterized statistics belong in the contract layer.
