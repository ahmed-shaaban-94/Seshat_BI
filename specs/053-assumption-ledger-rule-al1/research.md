# Phase 0 Research: Assumption Ledger Rule (AL1)

All items are grounded against the live tree on branch
`053-assumption-ledger-rule-al1`. No open technical unknowns remain.

## R1 -- Lazy-yaml contract-read seam (Principle II / B1 / B3)

**Finding**: `src/retail/metric_drift.py::load_definition()` imports `yaml` INSIDE
the function body ("lazy: dev/optional dep, kept out of the retail check core
chain") and reads the contract via `yaml.safe_load(Path(...).read_text(...))`. The
sibling reconciler rules (SC1/DF1) likewise do a lazy `import yaml` inside the rule
function to preserve the stdlib-only core import chain.

**Decision**: AL1 does a private lazy `import yaml` inside its rule function (not at
module scope) and reads each selected contract with
`yaml.safe_load(text)`. It reads the file text via the `RuleContext`
(`(ctx.repo_root / rel).read_text(encoding="utf-8-sig")`, matching PP1's tolerant
BOM-stripping read). It does NOT import `metric_drift` (which is a heavier module
not in the check core chain); it reuses the PATTERN, not the module, to avoid
dragging extra imports into the rule.

**Rejected alternative**: a module-scope `import yaml` -- would put
`retail.rules.assumptions` on a non-stdlib import at module load and trip B1/B3.

## R2 -- The generic contract SHAPE anchors AL1 keys off

**Finding** (from `templates/metric-contract.yaml` + the five instances under
`mappings/retail_store_sales/metrics/`):

- `readiness.status`: a string in `{not_started | blocked | warning | pass}`.
- `readiness.blocking_reasons`: a list; REQUIRED non-empty when status is `blocked`
  (per the template's own authoring notes).
- `binds_to.gold_table`: a string; a real binding is `gold.<fact_or_dim>`; the
  UNFILLED template form is the angle-bracket placeholder `gold.<fact_or_dim>` /
  `"<...>"`.
- `binds_to.columns`: a list of column names; the UNFILLED template form is a single
  `"<gold_column_a>"` placeholder entry.

**Decision**: AL1 keys ONLY on these generic field paths. The UNFILLED signal is the
`<...>` angle-bracket placeholder (the same convention G6 governs and PP1 reuses):
a `binds_to.gold_table` whose value still contains a `<...>` token, or a `columns`
list that is empty or whose every entry is a `<...>` placeholder, is treated as an
UNFILLED (not-yet-settled) binding. A `gold_table` with no angle brackets AND a
`columns` list with at least one non-placeholder entry is a SETTLED binding.

**Rejected alternative**: keying on a metric NAME or a specific gold table (e.g.
`gold.fct_sales_rss`) -- would inline a C086 specific and violate Principle VII.

## R3 -- Instance selection + exemption (Principle VII / template exclusion)

**Finding**: PP1 selects instances by suffix match over `ctx.tracked_files`,
excludes its generic template by exact path, and excludes test fixtures via
`is_test_path()`.

**Decision**: AL1 selects `p` from `ctx.tracked_files` where `p` matches
`mappings/*/metrics/*.yaml` (a path under some `mappings/<table>/metrics/` directory
ending in `.yaml`), EXCLUDES the exact generic template path
`templates/metric-contract.yaml`, and excludes any `is_test_path(p)`. The
`mappings/<table>` segment is matched generically (any table), never a literal
`retail_store_sales`.

**Rejected alternative**: a hard-coded `mappings/retail_store_sales/metrics/` prefix
-- C086 leak; excluded.

## R4 -- Registration / wiring / manifest contract

**Finding**: `@register("AL1", "<title>")` auto-wires the rule via the registry's
package discovery (no `registry.py` edit). `tests/unit/test_rules_wiring.py` asserts
the registered id set EQUALS `EXPECTED_RULE_IDS` (a frozenset) and that
`len(registry.all_rules()) == len(EXPECTED_RULE_IDS)` -- it keys off set length, not
a hard-coded integer. `docs/rules/rules-manifest.json` is the 043 golden snapshot,
regenerated by `retail manifest --repo .` and guarded by
`tests/unit/test_rules_manifest_snapshot.py`.

**Decision**: The same change that adds `assumptions.py` adds `"AL1"` to
`EXPECTED_RULE_IDS` (33 -> 34, AL1 the sole addition) and regenerates the manifest.
No literal count is written anywhere.

## R5 -- Wiring-latent-gap obligation

**Finding** (repo memory, G6 wiring latent gap): a rule can be registered and listed
in `EXPECTED_RULE_IDS` yet never actually exercised firing -- a silent-unvalidated
gap. The fix is a test that drives the rule over a known-bad fixture and asserts a
Finding is produced.

**Decision**: `tests/unit/test_assumptions.py` MUST include a test that runs AL1 over
a synthetic `blocked`+filled-binding contract fixture and asserts exactly one ERROR
Finding -- not merely that `AL1` appears in the registry.

## R6 -- Principle-V read-only boundary + standalone resolution

**Finding**: The Clarifications (Session 2026-07-01) resolved: marker = existing
`readiness.status: blocked` + non-empty `blocking_reasons` (C1); trigger = that
marker coexisting with a filled `binds_to` (C2); AL1 ships STANDALONE (C3), not
gated on the unshipped T1.2 define-half. The on-main contracts
(`mappings/retail_store_sales/metrics/*.yaml`) are `status: pass` (verified:
TotalSales and DiscountedTransactionRate both `pass`, empty `blocking_reasons`), so
none present the blocked+bound contradiction.

**Decision**: AL1 only READS the fields and emits a Finding; NO code path writes,
clears, resolves, or auto-answers the assumption or edits any readiness state
(FR-008). The zero-AL1-findings-on-main baseline is a GENUINE pass: the convention
exists and is checked; the tree simply has no offending contract. The governance
MEANINGS of C1/C2 are recorded to open_for_human for optional human override; the
build proceeds on the advisor defaults.
