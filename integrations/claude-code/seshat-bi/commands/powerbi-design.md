description: Route guarded Power BI dashboard and page design work
---

Load the `powerbi-workflows` skill and follow its design route. Data-bound
visual design requires approved metric contracts and committed semantic-model
evidence; without them, stop and name the missing gate instead of designing.
Before designing, use the installed read-only helpers when available:
`seshat dashboard-planner` for a new/extends/duplicate verdict on the proposal
and `seshat dashboard-gaps` for the pre-design inventory of design-blocking
gaps.
Produce reviewable design guidance only -- a layout plan, a visual list, a
filter rail (slicer fields and default selections), and a visual-to-contract
binding map. Never invent a metric, measure, or KPI meaning, never claim
dashboard readiness, and never mutate PBIP/PBIR files from this command.
