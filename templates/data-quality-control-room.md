# Data Quality Control Room -- portfolio roll-up

> GENERIC template. The portfolio-level sibling of the per-table
> `readiness-scorecard.md`: ONE consolidated, cross-table view of data-quality
> findings + process blockers, rendered by the `retail-control-room` skill from
> evidence that ALREADY EXISTS per table. It runs NO new check and edits NO per-table
> file (read-only). See `docs/readiness/readiness-model.md`.
>
> **Aggregates, never re-derives.** Every numeric cell is a MEASURED count copied from a
> committed source (a `retail check` WARN line, a `retail validate` ERROR, a
> `blocking-reasons.md` row). Never an adjective ("several", "mostly clean").
>
> **No fake confidence.** There is NO health/confidence score column, and a filled copy
> MUST NOT add one (roadmap rule #9). Readiness is the four explicit statuses + counts.
>
> **Generic, not C086.** Placeholder rows only; C086 is a cited filled instance, never
> inlined. ASCII only, UTF-8 no BOM.

## Portfolio roll-up (one row per table, worst-first)

Sort `blocked` above `warning` above `pass`. Every count is a measured number; cite its
source on request. A table with no per-table evidence is shown `not_started` / "no
evidence yet" -- never an invented status or count.

| Table id | Source family | Current stage | Status | Static WARNs | Live findings | Open blockers | Next action |
|----------|---------------|---------------|--------|--------------|---------------|---------------|-------------|
| `<table-a>` | `<source_family>` | `<stage_key>` | `blocked` | `<N>` | `<N>` | `<N>` | `<single next action from readiness-status.yaml>` |
| `<table-b>` | `<source_family>` | `<stage_key>` | `warning` | `<N>` | `<N>` | `<N>` | `<next action>` |
| `<table-c>` | `<source_family>` | `<stage_key>` | `pass` | `0` | `0` | `0` | `<next action>` |
| `<table-d>` | `<source_family>` | -- | `not_started` | -- | -- | -- | no evidence yet |

- **Static WARNs** = recorded `retail check` WARN lines for the table (`warning`).
- **Live findings** = recorded `retail validate` ERRORs (V-RC2 / V-RC15 / V-RC16,
  `error`). If not run recently: "not run since `<date>`".
- **Open blockers** = count of "Open blockers" rows in the table's `blocking-reasons.md`.

## Portfolio open-blockers (the action half, worst-first)

outranks a `warning` static WARN). Each field copied from the source `blocking-reasons.md`;
nothing invented. A blocker with no named owner is "UNASSIGNED" and flagged.

## Notes

- This view is read-only and point-in-time. A historical trend over time is the
  Reconciliation Ledger's job (F015), not this view.
- Clearing any blocker is a per-table action by its named owner (Principle V); the
  control room never clears, approves, or self-assigns.
